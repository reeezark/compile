## p1-p6 标题、编译阶段和目标代码生成主要任务

- Lecture 9：Target Code Generation。
- 目标代码生成位于优化后的 IR 到目标机器代码之间。
- 输入：源程序 IR 和符号表。
- 输出：目标代码，可能是汇编或机器码。
- 目标代码生成关注：
  - 如何让目标代码更短。
  - 如何充分利用寄存器、减少存储访问。
  - 如何利用目标机器指令系统特征。
- 三个主要任务：
  - Instruction selection：选取目标机器指令实现 IR。
  - Register allocation and assignment：决定哪些值放入哪些寄存器。
  - Instruction ordering/scheduling：决定指令执行顺序。
- 指令选择示例：`a=b+c; d=a+e` 转为 `LD/ADD/ST` 序列。
- 寄存器数量有限，寄存器分配和一般最优指令顺序选择都是困难问题，通常是 NP 相关问题。

## p7-p13 栈式计算机、累加器优化与 MIPS 基础

- 栈式计算机：
  - 无变量或显式寄存器。
  - 栈保存中间结果。
  - 指令从栈顶取操作数、弹出操作数、计算并压回结果。
- 栈顶访问非常频繁；加法原本需要两次读和一次写。
- 累加器 accumulator：把栈顶保存在寄存器中，使 `add` 变成 `acc <- acc + top_of_stack`，只需一次内存访问。
- 示例 `3+7+5` 展示栈式求值过程。
- MIPS 模拟栈式机器：
  - ACC 放在 `$t0`。
  - 栈放在内存。
  - `$sp` 保存下一可用栈位置，栈顶在 `$sp+4`。
  - 栈向低地址增长。
- MIPS 是典型 RISC：
  - 算术操作只使用寄存器。
  - 内存数据必须通过 `lw`/`sw` 读写。
  - 32 个 32-bit 通用寄存器。
- 指令：
  - `lw reg1 offset(reg2)`
  - `add reg1 reg2 reg3`
  - `sw reg1 offset(reg2)`
  - `addiu reg1 reg2 imm`
  - `li reg imm`

## p14-p20 寄存器保存恢复、栈操作、表达式和条件代码生成

- 临时变量无限，但物理寄存器有限，必须复用寄存器。
- 复用寄存器前要保存，之后再恢复。
- 保存 `$t0` 到栈：
  - `sw $t0, 0($sp)`
  - `addiu $sp, $sp, -4`
- 恢复栈顶值到 `$t0`：
  - `lw $t0, 4($sp)`
  - `addiu $sp, $sp, 4`
- 压栈多个寄存器时，先移动 `$sp` 为新数据留空间，再 `sw`。
- 示例 `7+5` 的 MIPS 翻译：
  - `li $t0 7`
  - 保存 7 到栈并移动 `$sp`
  - `li $t0 5`
  - `lw $t1 4($sp)`
  - `add $t0 $t0 $t1`
  - `addiu $sp $sp 4`
- 弹栈只移动 `$sp`，内存中旧数据仍存在但视为无效。
- 代码生成函数 `cgen(e)`：
  - 生成代码后表达式结果放在 `$t0`。
  - 保持 `$sp` 和栈内容一致。
- `cgen(e1+e2)`：先算 `e1` 并压栈，再算 `e2`，弹出 `e1` 到 `$t1`，执行 `add $t0 $t1 $t0`。
- 条件代码生成新增：
  - `beq reg1 reg2 label`
  - `b label`
- `if e1==e2 then e3 else e4`：先比较 `e1/e2`，相等跳 true_branch，否则执行 false_branch，最后跳到 end_if。

## p21-p24 运行时内存布局、活动记录和 AR 内容

- 内存区域：
  - Code：目标代码，编译期大小固定。
  - Global/static：全局常量等大小编译期已知的数据对象。
  - Stack：动态数据结构，尤其是调用栈和活动记录。
  - Heap：长生命周期数据，由运行时动态管理。
- 活动 activation：一次过程/函数执行。
- 活动记录 AR：管理单次过程执行所需信息。
- 栈保存过程调用产生的 AR。
- AR 栈管理：
  - 函数入口：在栈顶分配 AR。
  - 函数返回：从栈顶移除 AR。
- `$SP`：栈指针，指向栈顶，移动 `$SP` 即可分配/释放栈空间。
- `$FP`：帧指针，保存当前帧基地址；变量地址通常翻译为相对 `$FP` 的偏移。
- AR 常见内容：
  - 临时变量。
  - 局部变量。
  - 保存的寄存器。
  - 调用者返回地址/指令指针。
  - 调用者 AR 帧指针。
  - 参数。
  - 返回值。

## p25-p30 调用者/被调用者约定与函数调用代码生成

- 函数调用会覆盖寄存器，必须决定谁保存/恢复。
- Caller knows：哪些寄存器对调用者重要。
- Callee knows：自己会使用和覆盖哪些寄存器。
- 解决方式：调用者和被调用者合作。
- Caller-saved：
  - `$t0-$t9`, `$a0-$a3`, `$v0-$v1`
  - 调用者若关心这些值，要在调用前保存，调用后恢复。
- Callee-saved：
  - `$s0-$s7`, `$ra`
  - 被调用者若使用这些寄存器，要负责保存和恢复。
- 调用者步骤：
  - 保存关心的 caller-saved 寄存器。
  - 参数按从右到左顺序压栈。
  - 跳转到函数，保存下一条指令地址到 `$ra`。
- 被调用者步骤：
  - 保存旧 `$fp` 和 `$ra`。
  - 设置自己的 `$fp`。
  - 为局部变量和临时变量分配栈帧空间。
  - 执行函数体。
- 返回步骤：
  - 返回值放入 `$v0`。
  - 弹出栈帧，恢复 `$fp` 和 `$ra`。
  - `jr $ra` 跳回调用点。
  - 调用者弹出参数并恢复 caller-saved 寄存器。
- 新指令：
  - `jal label`：跳到 label，并把下一条指令地址保存到 `$ra`。
  - `jr reg`：跳到寄存器保存的地址。

## p31-p33 变量访问与 FP 偏移

- 函数变量（参数、局部变量、临时变量）都在 AR 中。
- 栈会随中间结果保存而增长/收缩，因此变量不能可靠地用 `$sp` 相对地址访问。
- `$fp` 在当前函数执行期间保持稳定，适合通过固定偏移访问变量。
- 参数由调用者从右到左压栈。
- 局部变量由被调用者分配。
- 示例：
  - `$fp` 指向旧返回地址/旧 IP 附近。
  - `x: +8($fp)`
  - `y: +12($fp)`
  - 第一个局部变量：`-4($fp)`
- `fun1/fun2` 示例展示嵌套调用时不同 AR 在栈中并存，各自有自己的 `$fp/$sp`。

## p34-p39 面向对象代码生成：对象布局、静态/动态分派、继承

- 对象类似 C 结构体：
  - 对象连续存放。
  - 成员变量在对象内有固定 offset。
  - 对象额外有成员方法。
- 非虚方法：
  - 不能被覆盖。
  - 调用目标由静态引用类型决定。
  - 编译器可在编译期确定目标，即 static dispatch。
- 虚方法：
  - 可被子类覆盖。
  - 调用目标由运行时对象类型决定。
  - 需要 dynamic dispatch。
- Dispatch：
  - 静态分派：编译期决定函数地址，可硬编码到二进制。
  - 动态分派：运行时决定函数地址。
- 动态分派实现：
  - 编译期为每个类生成 dispatch table，包含所有虚方法目标地址。
  - 运行时每个对象包含指向 dispatch table 的指针。
  - 方法调用通过 dispatch pointer 加偏移找到真实目标。
- 典型对象布局：
  - Class tag：动态类型检查。
  - Dispatch ptr：指向 dispatch table。
  - 成员变量 `x/y/z`。
- 继承布局不变式：
  - 成员变量和成员方法在父类及所有子类中的 offset 必须一致。
  - 子类新增字段/方法追加在后面。
  - 覆盖方法时在同一 dispatch table offset 替换函数地址。

## p40-p49 机器相关优化：指令选择、成本、调度、寄存器分配、窥孔优化

- IR 优化后，需要转为目标语言，考虑具体架构特征：
  - 专用指令。
  - 硬件流水线。
  - 寄存器细节。
- 典型机器相关优化：
  - Instruction selection and scheduling。
  - Register allocation。
  - Peephole optimization。
- 指令选择：
  - 找到 IR 到目标汇编的高效映射。
  - CISC 架构如 x86 中，同一 IR 操作可能有多种指令实现，如 `a=a+1` 可用 `ADD #1` 或 `INC`。
- 指令成本：
  - `Instruction cost = 1 + cost(source-mode) + cost(destination-mode)`。
  - 示例展示不同实现成本可从 6 降到 2。
- 指令调度：
  - 指令有 latency。
  - 现代机器可每周期发射多条操作。
  - 调整指令顺序以减少等待、避免溢出、提升局部性。
- 寄存器分配：
  - TAC 有无限变量，物理机器寄存器有限。
  - 目标：高频访问值放寄存器；只在变量活跃期间占用寄存器。
  - 局部分配：按基本块决策。
  - 全局分配：跨基本块保持映射一致，并考虑 CFG。
  - 常见算法：图着色、线性扫描、整数线性规划。
- 图着色：
  - RIG 节点代表变量。
  - 边代表 live ranges 干涉。
  - 无边变量可共享寄存器，有边变量不能共享。
  - k 可着色即 k 寄存器可分配。
- 寄存器溢出：
  - k-coloring 是 NP-complete。
  - 实践用启发式算法，如 Chaitin graph coloring。
  - 若无法分配，则把变量 spill 到内存，再重试。
  - Spilling 慢，因为需要 load/store。
- 窥孔优化：
  - 用滑动窗口检查少量目标指令。
  - 把窗口内指令序列替换为更短或更快序列。
  - 可用于冗余指令消除、控制流优化、代数化简。
  - 示例：jump to jump 可改为直接跳最终目标。

## p50 总结

- 代码可在不同层级优化：
  - Peephole、local、loop、global。
  - IR 层：local/global/CSE/constant folding/constant propagation。
  - Target 层：instruction/register 等。
- 优化技术之间会相互影响：
  - 一个变换可能暴露另一个优化机会。
  - 一个优化也可能隐藏或移除另一个机会。
- 最优优化组合仍是研究问题。
- 编译器通常把常一起使用的优化打包为等级，如 `-O1`、`-O2`。
