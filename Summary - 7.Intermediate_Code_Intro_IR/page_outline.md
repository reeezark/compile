## p6-p9 多层 IR

- IR 的优势：更高抽象、更清晰阶段分离。
- 现代编译器可使用一系列 IR。
- High-level IR：接近源语言，如 Parse Tree、AST；语言相关，用于语义分析。
- Low-level IR：接近汇编，如 TAC、SSA；抽象机器指令集，语言和机器相对独立，用于机器无关优化。
- Machine-level IR：接近具体 ISA，如 x86 IR、ARM IR；机器相关，用于寄存器分配和目标代码生成。
- 可以只使用 AST 一种 IR，适合编译时间优先的场景如 JIT；多层 IR 更利于任务匹配和前后端复用。

## p10-p18 中间表示与三地址码

- 重要 IR：树形结构、DAG、线性表示，尤其是三地址码。
- 三地址码 TAC 的通用形式：`X = Y op Z`，右侧最多一个操作符。
- 特性：语法树/DAG 的线性化，抽象机器汇编，长表达式拆成多条指令，控制流转成跳转，机器无关。
- 常见地址：源程序名字、常量、编译器生成临时变量。
- 三地址指令形式：
  - 二元赋值、一元赋值、复制。
  - 无条件跳转、条件跳转。
  - 参数传递、过程调用、返回。
  - 数组下标复制、地址/指针赋值。
- `do { i=i+1; } while(a[i]<v)` 示例展示数组访问转地址计算和跳转。

## p19-p30 TAC 实现：四元式、三元式、间接三元式

- 三地址码可用四元式、三元式、间接三元式等实现，取舍涉及空间、速度和可操作性。
- 四元式：四个字段 `op, arg1, arg2, result`。
- 一元操作、param、goto、条件跳转等字段可能为空或把目标 label 放入 result。
- 示例 `a = b * (-c) + b * (-c)` 的四元式。
- 三元式：字段 `op, arg1, arg2`，结果隐含为指令编号。
- 三元式优点是少一个 result 字段；缺点是优化中移动指令会导致所有基于编号的引用需要更新。
- 复杂左值如 `x[i]=y` 可能需要两条三元式：先计算地址，再赋值。
- 间接三元式：三元式存入数据库，真正执行序列是指向三元式的指针列表；移动指令只需重排指针列表。
- CSE 后可复用三元式数据库中的空洞。

## p31-p33 SSA

- SSA：Single Static Assignment，每个变量静态上只赋值一次。
- 每次赋值使用不同版本名，如 `x1, x2, ...`。
- 控制流合并处使用 φ 函数，如 `x5 = φ(x3, x4)` 表示根据控制流选择不同版本。
- SSA 优化收益：
  - 告诉编译器什么时候不能做 CSE，避免把不同值误认为相同。
  - 告诉编译器什么时候应该做 DCE，死值更明显。
  - 显式表达值流，不必额外构造数据流图。

## p34-p43 语法制导代码生成与变量布局

- 语法制导翻译可用于代码生成：根据 AST/语法结构生成 TAC。
- 需要翻译的结构：定义、赋值、数组引用、布尔表达式、控制流语句。
- 策略：先写无顺序 SDD 语义规则，再转为有执行顺序 SDT 动作。
- 函数代码生成两步：
  - 处理变量定义，完成内存布局。
  - 处理语句，生成 TAC。
- 变量布局需要 location 和 width。
- 属性：`T.type/T.width`、`V.type/V.width/V.offset`。
- naive 全局大区间分配浪费内存；更好方式是按作用域分配，进入作用域分配，退出作用域释放。
- 函数局部变量按当前 offset 顺序布局。
- 内存对齐：要求地址按类型大小边界对齐；未对齐会影响机器访问效率。
- `struct { char x; int y; }` 未对齐大小为 5，对齐后通常为 8。

## p44-p46 赋值语句翻译

- 表达式中多操作符被拆成右侧最多一个操作符的 TAC。
- 辅助函数：
  - `lookup(id)` 查符号表。
  - `emit()/gen()` 生成三地址码。
  - `newtemp()` 生成新临时变量。
- 属性 `code` 表示产生的代码，`addr` 表示表达式值所在地址。
- 示例 `a = b + (-c)` 生成：
  - `t1 = minus c`
  - `t2 = b + t1`
  - `a = t2`
- 增量翻译：`gen()` 不仅构造指令，也追加到已有指令序列。

## p47-p52 数组引用翻译

- 数组引用核心是确定元素地址。
- 一维数组：`addr(A[i]) = base + i * width`。
- 二维数组按行主序：`addr(A[i1,i2]) = base + i1*N2*width + i2*width`。
- k 维数组：`addr(A[i1]...[ik]) = base + i1*W1 + ... + ik*Wk`。
- 示例：
  - `array(10,int)`：`a[i]` 偏移为 `i*4`。
  - `array(3,array(5,int))`：`a[i1][i2]` 偏移为 `i1*20+i2*4`。
  - `array(3,array(5,array(8,int)))`：`a[i1][i2][i3]` 偏移为 `i1*160+i2*32+i3*4`。
- 属性：
  - `L.array` 指向数组符号表条目。
  - `L.type` 表示当前子数组类型。
  - `L.addr` 表示已计算出的偏移临时变量。
- 对 `c = a[i1][i2][i3]` 生成逐步计算偏移和取值的 TAC。

## p53-p58 布尔表达式翻译

- 布尔表达式包含关系运算和布尔运算：`<, <=, !=, >, >=, &&, ||, ==` 等。
- 短路求值：
  - `flag || foo()` 中若 `flag` 为真，不调用 `foo()`。
  - `flag && foo()` 中若 `flag` 为假，不调用 `foo()`。
- 可把布尔值先当普通表达式算出临时变量，再用于条件跳转。
- 更适合控制流的方式是直接翻译为跳转：每个布尔表达式有 `E.true` 和 `E.false` 两个出口。
- `||`：左操作数真则整体真，左操作数假才评估右操作数。
- `&&`：左操作数假则整体假，左操作数真才评估右操作数。
- `!`：交换 true/false 出口。
- `true/false`：直接生成到对应出口的 goto。

## p59-p70 控制流翻译与回填

- 控制语句包括 `if(B)S1`、`if(B)S1 else S2`、`while(B)S1`。
- 继承属性：
  - `B.true`：B 为真时跳转目标。
  - `B.false`：B 为假时跳转目标。
  - `S.next`：语句 S 之后的下一条指令标签。
- 辅助函数：
  - `newlabel()` 创建新标签。
  - `label(L)` 把标签 L 贴到下一条 TAC。
- 关键问题：前向跳转目标尚未生成。
- 两遍法：第一遍生成伪标签并记录，第二遍回填。
- 单遍法：生成空洞，维护 hole list，目标地址已知时回填。
- Backpatching 属性：
  - `B.truelist`：B 为真时待回填的跳转列表。
  - `B.falselist`：B 为假时待回填的跳转列表。
  - `S.nextlist`：跳到 S 后继位置的待回填列表。
- 辅助函数：
  - `makelist(i)` 创建单元素列表。
  - `merge(p1,p2)` 合并列表。
  - `backpatch(p,i)` 把列表中所有空洞填成目标 i。
- 使用 `M -> ε` 记录 `nextinst`，使用 `N -> ε` 生成跳过 else 的 `goto _`。
